java中的transient关键字

Posted on 2019-02-04

transient用法

  • 被transient修饰的变量不会被序列化或写到磁盘中。
  • 一旦对象被transient修饰，变量将不再是对象持久化的一部分，该变量内容无法在序列化后被访问。
  • 该方法只能够修饰变量，而不能够修饰方法和类。

注：实现序列化的方法是通过实现Serializable接口。

transient 踩坑

一个静态变量无论是否被transient修饰，均不能序列化。

不能够序列话的变量并不是说存储时，存储的null，而是什么都不进行存储。

并不一定被transient修饰的变量就一定不能被序列化

在Java中，对象的序列化可以通过两种接口来实现，若实现的Serializable接口，所有的序列化会自动进行，但是如果实现的是Externalizable接口，则没有任何东西可以自动进行序列化，需要在writeExternal中指定所要序列化的变量，这与是否被transient修饰无关。

java 抽象类与接口的区别

Posted on 2019-02-01

抽象类

包含抽象方法的类叫做抽象类。如果一个类包含一个或多个抽象方法，该类必须限定为抽象的。

什么是抽象方法

抽象方法即不完整的方法，仅有声明而没有方法体且必须使用关键字abstract做修饰。

抽象类中可以没有抽象方法，但是有抽象方法的类一定是抽象类

当一个子类要继承抽象类时，如果子类还相当抽象类就不一定要实现所有的抽象方法、如果子类不愿再当抽象类了就必须实现父类所有的抽象方法

抽象类不能被实例化，只能被继承，其中抽象方法只能由子类来继续实现。

悲观锁与乐观锁

Posted on 2019-01-31

悲观锁与乐观锁

什么是悲观锁与乐观锁

乐观锁对应于生活中乐观的人总是想着事情往好的方向发展，悲观锁对应于生活中悲观的人总是想着事情往坏的方向发展。这两种人各有优缺点，不能不以场景而定说一种人好于另外一种人。

悲观锁

总是假设最坏的情况，每次拿数据时都认为别人会修改，所以每次拿数据时都会上锁，这样别人想拿这个数据就会阻塞知道它拿到锁（共享资源每次只给一个线程使用，其他线程阻塞，用完之后再把资源转让给其他线程）。传统的关系型数据库中便是用到了这种锁机制，比如：行锁，表锁等，读锁，写锁等，都是在操作之前先上锁。java中的synchronized和Reentrantlock等独占锁都是悲观锁思想的实现。

乐观锁

总是假设最好的情况，每次拿数据时都认为别人不会修改，所以不会上锁，但是在更新的时候会判断以下在此期间别人有没有更新这个数据，可以使用版本号机制和CAS算法实现。乐观锁适用于多读的应用类型，这样可以提高吞吐量，像数据库提供的类似于write_condition机制，其实都是提供的乐观锁。在java中java.util.cocurrent.atomic包下面的原子变量类就是使用了乐观锁的一种实现方法CAS实现的。

JSP

JSP简介

扩展名 .jsp
互联网媒体类型 application/jsp
最新版本 2.2
标准 JSR 245
网站 JavaServer Pages Technology

JSP（全称JavaServer Pages）是由Sun Microsystems公司主导创建的一种动态网页技术标准。JSP部署于网络服务器上，可以响应客户端发送的请求，并根据请求内容动态地生成HTML、XML或其他格式文档的Web网页，然后返回给请求者。JSP技术以Java语言作为脚本语言，为用户的HTTP请求提供服务，并能与服务器上的其它Java程序共同处理复杂的业务需求。

JSP将Java代码和特定变动内容嵌入到静态的页面中，实现以静态页面为模板，动态生成其中的部分内容。JSP引入了被称为“JSP动作”的XML标签，用来调用内建功能。另外，可以创建JSP标签库，然后像使用标准HTML或XML标签一样使用它们。标签库能增强功能和服务器性能，而且不受跨平台问题的限制。JSP文件在运行时会被其编译器转换成更原始的Servlet代码。JSP编译器可以把JSP文件编译成用Java代码写的Servlet，然后再由Java编译器来编译成能快速执行的二进制机器码，也可以直接编译成二进制码。

实际上JSP就是对Servlet的包装。

java web 简介

Posted on 2019-01-14

web简史

静态页面

静态页面指的是用户请求访问的界面都是事先编辑好的，不能改变的。静态页面如何提交请求并获得访问结果。

  • 客户代理（一般指浏览器）发起资源请求。
  • 服务器收到请求后，根据URL查找对应目录下的资源文件，然后构造http响应将资源文件返回给客户代理。

CGI

CGI是一个可执行程序或可执行脚本。几乎所有的语言都可以写CGI。CGI的工作流程如下。

  • 客户代理请求服务。
  • http服务器启动对应的CGI获得结果
  • 服务器将结果返回服务器

对于服务器来说，不同的服务对应着不同的CGi，也就是说，用户每申请一个请求，就得启动一个CGI，对资源的消耗非常巨大。

什么是Servlet

Java Servlet是运行在Web服务器或应用服务器上的程序，它是作为Web浏览器或其他HTTP客户端的请求和HTTP服务器上的数据库或应用程序之间的中间层。

使用Servlet通常情况下，可以达到与CGI（Common Gateway Interface，公共网关接口）实现的程序可以达到相同的效果。但是相比与CGI，servlet有更为明显的优势：

  • 性能更优
  • Servlet在Web服务器的地址空间内执行。这样就不必要创建一个单独的进程来处理每个客户端请求。
  • Servlet独立于平台，使用Java编写。
