计算机网络-网络层

Posted on 2018-11-08 | Edited on 2018-11-09

前言：

  • 两种用于构造网络层分组交付的方法，即数据包模式和虚电路模式
  • 编址
  • 网络层的转发($forwarding$)功能和路由选择($routing$)功能。转发涉及分组在单一的路由器中从一条入链路到一条出链路的传送。路由选择涉及一个网络的所有路由器，它们经路由选择协议共同交互，以决定分组从源到目的地节点所采用的路径。
  • 网际协议($ IP $)。网络层编址和$IPv4$的数据报格式。
  • 网络地址转换($NAT$)、数据报分段、因特网控制报文协议（$ICMP$）和$IPv6$。
  • 网络层的路由选择功能：链路状态算法和距离矢量算法。

概述

  • 网络层实现的是主机与主机之间的通信。

  • 网络层在计算机网络中为运输层提供支持：

    • 运输层实现进程到进程的通行

    • 运输层功能的实现依赖于网络层提供的服务

  • 在数据报文传输的路径上，每一台分组交换机都需要实现网络层以完成数据分组交付

  • 网络层的主要功能：

    • 在全局范围内为主机间通信选路，选路的结果反映为分组交换机上的转发表。

    • 分组交换机上的网络层根据转发表以及分组头部信息，将分组向适当的链路进行转发。

    • 对于面向连接的网络层服务，提供建立连接的功能。

转发和路由选择

网络层提供的功能看似极为简单：将分组从一台发送主机移动到一台接收主机。为此，需要两种重要的网络层功能：

  • 转发。当一个分组到达路由器的一条输入链路时，路由器必须将改分组移动到合适的输出链路。

  • 路由选择。当分组从发送方流向接收方时，网络层必须决定这些分组所采用的路由或路径。计算这些路径的算法被称为路由选择算法$(rounting \quad algorithm)$。

每台路由器都有一张转发表($forwardning \, table$)。转发表是由路由选择算法计算得出的。路由选择算法可能是集中式的，或是分布式的

分组交换机是指一台通用分组交换设备。它根据分组首部字段中的值，从输入链路到输出链路接口转移分组。某些分组交换机称为链路层交换机$(link-layer switches)$，基于链路层字段中的值做转发决定。其他分组交换机称为路由器(router)。

网络层除了转发和路由选择两个重要功能外，还有第三个重要的功能，即连接建立（$connection\,setup$）。

中位数与顺序统计量

在一个由$n$个元素组成的集合中，第$i$个顺序统计量（$ order statistic $）是该集合中第$i$小的元素。例如，在一个元素集合中，最小值是第1个顺序统计量$(i = 1)$，最大值是第$n$个顺序统计量$(i = n)$。

中位数是它所属集合的“中点元素”。当$n$为奇数时，中位数是唯一的。当$n$为偶数时，存在两个中位数，分别位于$i = n/2$和$i = n/2 + 1$处。

接下来我们讨论一个由$n$个互异的元素构成的集合中选择第$i$个顺序统计量的问题。对于这一问题形式化定义为如下的选择问题:

<strong>输入：</strong>一个包含$n$个（互异的）数的集合$A$和一个整数$i, 1 \leq i \leq n$。

<strong>输出：</strong>元素$x \in A$，且$A$中恰好有$i - 1$个其他元素小于它。

通过选择合适的排序算法，我们可以在有限时间内快速解决这个问题(堆排序或归并排序$O(lgn)$)。这里学习一些更快的算法。

最大值和最小值问题

对于一个具有$n$个元素的数组，我们可以在比较最多$n-1$次的情况下，寻找到最小值或最大值。一种更优的算法同时寻找最大值和最小值。

<strong>    同时寻找最大值和最小值</strong>

如果独立的去寻找最大值和最小值，需要$(n - 1) + (n - 1) = 2n - 2$次比较。事实上，我们可以最多$3\times \lfloor n/ 2 \rfloor$次比较就可以同时寻找最大值和最小值。具体的如下：每次对两个数进行比较，将较大的值与已知的最大值比较，将较小值与已知的最小值比较。这样每次我们仅仅需要3次比较，而总共需要比较的数对有$n/2$对，所以总共的比较次数为$3\times \lfloor n/ 2 \rfloor$。

对于如何得到最初的最大值与最小值呢？实际上开始的最大值与最小值的设定依赖于$n$是偶数还是奇数，当$n$为奇数时，最大值与最小值相等，均为第一个数值。当$n$为偶数时，将前两个元素进行一次比较，确定最大值与最小值的初始值。

期望为线性时间的选择算法

接下来我们要学习一种解决选择问题的分治算法。$RANDOMIZED-SELECT$是以快速排序算法为模型的。该算法利用了$RANDOMIZED-PARTITION$过程。

该算法使用随机分割将待寻找部分分割为2部分，其中该数组中比分割点小的元素，移动到分割点之前；该数组中比分割点大的元素，移动到分割点之后。之后进行下一次寻找。期望时间复杂度$O(n)$，最坏时间复杂度$O(n……2)$.

最坏时间为线性时间的选择算法

现在我们来看一个最坏情况运行时间为$O(n)$的选择算法：$SELECT$。和$RANDOMIZED-SELECT$一样，$SELECT$算法也是通过对输入数组的递归划分来找出所需元素，但是，在该算法中能够保证对数组的一个好的划分。$SELECT$算法使用的是来自快速排序的确定性划分算法$PARTITION$，但做出了修改，将划分的主元也作为输入参数。

通过执行下列步骤，算法$SELECT$可以确定一个有$n>1$个不同元素的输入数组中第$i$小的元素。（如果$n = 1$，则$SELECT$只返回它的唯一输入数值作为第$i$小的元素。）

  1. 将输入数组的$n$个元素划分为$\lfloor n/5 \rfloor$组，每组5个元素，且至多只有一组由剩下的$n mod 5$个元素组成。
  2. 寻找这$\lceil n/5 \rceil$组中每一组的中位数：首先对每组元素进行插入排序，然后确定每组有序元素的中位数。
  3. 对第二步中找到的$\lceil n/5 \rceil $个中位数，递归调用$SELECT$以找出其中位数$x$（如果有偶数个中位数，为了方便，约定$x$是较小的中位数）。
  4. 利用修改过的$PARTITION$版本，按中位数的中位数$x$对输入数组进行划分。让$k$比划分的低区中的元素数目多1，因此$x$是第$k$小的元素，并且有$n-k$个元素在划分的高区。
  5. 如果$i = k$，则返回$x$。如果$ik$则在高区中递归查找第$i-k$小的元素。

$$ n = i_1 + i_2 + \cdot\cdot\cdot + i_k $$

动态规划(Dynamic Programming)

动态规划是与分治法类似的一种计算方法，都是通过组合子问题的解来求解原问题。

动态规划常常适用于有重叠子问题和最优子结构性质的问题，动态规划方法所耗时间往往远少于朴素解法。

动态规划方法通常用来求解最优化问题(optimization problem)。这类问题可以有很多可行解，每个解都有一个值，我们希望寻找具有最优值（最大值或最小值）的解。

动态规划将一个大问题分解为小问题（子问题），再根据子问题的解以得出原问题的解，与分治法不同的是，动态规划方法尝试在计算过程中将子问题的解记录下来，以便下次遇到相同的子问题时可以直接获取答案。

适用范围

动态规划在查找有重叠子问题的问题的解时有效。它将问题划分为子问题，分别求解并记录子问题的解，再将子问题的解组合起来。因此，动态规划在遇到具有相同子问题时不会耗费时间。

动态规划只能应用于有最后子结构的问题。最优子结构的意思是局部最优解能够决定全局最优解（并不一定能满足，有时需要近似）。

无后效性。即子问题的解一旦确定，就不再改变，不再受这之后、包含它的更大的问题的求解决策影响。

实例

  • 钢条切割
  • 斐波那契数列
  • 背包问题

  • 最长公共子序列

分治法

算法思想

将原问题分解为几个规模较小大但类似于原问题的子问题，递归的求解这些子问题，然后再合并这些子问题的解来建立原问题的解。

分治模式在每层递归时都有三个步骤：

  • 分解原问题为若干子问题，这些子问题是原问题的规模较小的实例。
  • 解决这些子问题，递归地求解各子问题。然而，若子问题的规模足够小，则直接求解。
  • 合并这些子问题成原问题的解

归并排序算法完全遵循分治模式。直观上其操作如下：

  • 分解：分解待排序的n个元素的序列成各具$n/2$个元素的两个子序列。

  • 解决：使用归并排序递归地排序两个子序列。

  • 合并：合并两个已排序的子序列以 产生已排序的答案。

    当待排序的序列长度达到可分解的最小单元长度时，递归“开始回升”。

    归并排序算法的关键操作是“合并”步骤中两个已排序序列的合并。
