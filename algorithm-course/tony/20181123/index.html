排序算法

对于排序问题，我们给出这样的一般性形式化定义：

输入：$n​$个数的一个序列<$a_1, a_2, a_3,\cdots a_n​$>。

输出：输入序列的一个排列<$a_1^，, a_2^，, \cdots, a_n^，$>，满足<$a_1^，\leq a_2^，\leq \cdots a_n^，$>。

一个定理

任何基于比较的排序算法都需要做$nlgn$次比较。即，运算下界是$O(nlogn)$.

一个推论

堆排序和归并排序都是渐进最优的比较排序算法。

插入排序(Insertion Sort)

我们开始讲解的第一个排序算法是插入排序。

对于少量元素，插入排序是一个非常有效的算法。插入排序的执行过程就像是打牌过程中，我们每抽到一张牌，便寻找合适的位置将它插进去。问题在于，我们寻找合适位置的依据。

算法原理：数组分为两部分，已排序集合$Q$，未排序集合$P$，每次检查集合$P$是否为空，为空则结束算法运行，否则从集合$P$中选择该集合中的任何一个元素尝试插入到有序集合$P$中。

算法过程：

  1. 从第一个元素开始，该元素可以认为已经被排序
  2. 取出下一个元素，在已经排序的元素序列中从后向前扫描
  3. 如果该元素（已排序）大于新元素，将该元素移到下一位置
  4. 重复步骤3，直到找到已排序的元素小于或者等于新元素的位置
  5. 将新元素插入到该位置后
  6. 重复步骤2~5

对于插入排序，我们通常使用的是原址排序$in-place-sort$。

很显然，在将元素插入到已排序数组中的过程需要将其后的元素移动1个单位。（移动元素的消耗）

时间复杂度分析：

  • 遍历耗时：n个元素，全部访问一遍。
  • 插入耗时：每个元素要插入到已排好序的序列中
    • 最好情况：不移动，即该元素是已排好序序列中最大的
    • 最坏情况：移动$k-1$次（该元素下标为k）。
    • 平均情况：$k/2$。
  • 时间复杂度：$O(n^2)$

空间复杂度分析：原址排序，$O(1)$.

冒泡排序(Bubble Sort)

冒泡排序这个名字的由来是：在排序的过程中，约小的元素（或越大的元素）经过交换会慢慢的“浮”到数组的顶端。冒泡排序也是一种$in-place$排序。

冒泡排序的执行过程：

  1. 比较相邻的元素。如果第一个比第二个大，就交换他们两个。
  2. 对每一对相邻元素作同样的工作，从开始第一对到结尾的最后一对。这步做完后，最后的元素会是最大的数。
  3. 针对所有的元素重复以上的步骤，除了最后一个。
  4. 持续每次对越来越少的元素重复上面的步骤，直到没有任何一对数字需要比较。

执行过程中我们发现：每次执行都会挑出一个最大的元素（或者最小的元素放在末尾）。

时间复杂度分析：

  • 最好情况：输入为升序排列，需要比较$n^2$次，但是不移动任何一个元素(作为优化，如果发现在一趟比较中没有移动任何元素，则可以断言该输入已经有序，为什么？如果有序则不需要再进行下去了，那么时间复杂度为$n$，为什么？)
  • 最坏情况：输入为降序排列，需要比较$n^2$次。
  • 平均：$n^2$.

空间复杂度：

  • 只需要额外的一个空间用于交换，$O(1)$.

桶排序(Bucket sort)

桶排序又称箱排序，工作原理是将待排序元素分到数量有限的桶中。每个桶分别排序。桶排序不是基于比较的排序，故不受比较下界的影响。

桶排序的执行过程：

  1. 设置一个定量的数组当作空桶子。
  2. 寻访序列，并且把项目一个一个放到对应的桶子去。
  3. 对每个不是空的桶子进行排序。
  4. 从不是空的桶子里把项目再放回原来的序列中。

在进行每个桶的排序的时候，我们可以选择任意的排序算法（其他算法，或递归对每个桶排序）进行排序。

值得注意的是，算法中的一个条件是：数量有限。所以，我们在实际应用中，何时用到桶排序呢？

如果，待排序数组的数据范围有限（类似于计数排序）即：对于任意的元素$x,\text{有}m\leq x \leq k$.可以设立k个桶（注意该处消耗的空间。

快速排序(Quick sort)

对于快速排序来说，这是一种最快请开个时间复杂度为$\Theta(n^2)$的排序算法，虽然最坏情况非常差，但是它的平均性能非常好：期望的运行时间是$\Theta(n\, lgn)$，并且$\Theta(n\, lgn)$中隐含的常熟因子非常小，此外它还是一种原址排序

与归并排序一样，快速排序依然使用了分治的思想。下面是对一个典型的子数组$A[p \cdots r]$进行快速排序的三步分治过程。

分解:数组$A[p \cdots r]$被划分为两个（可能为空）子数组$A[p\cdots q-1]$和$A[q+1 \cdots r]$ ，使得$A[p\cdots q-1]$ 中的每一个元素都小于等于$A[q]$，而$A[q]$也小于等于$A[q+1 \cdots r]$中的每个元素。其中，计算下标q也是划分过程的一部分。

解决:通过递归调用快速排序，对子数组$A[p\cdots q-1]$和$A[q+1 \cdots r]$进行排序。

合并:因为子数组都是原址排序的，所以不需要合并操作：数组$A[p\cdots r]$已经有序
