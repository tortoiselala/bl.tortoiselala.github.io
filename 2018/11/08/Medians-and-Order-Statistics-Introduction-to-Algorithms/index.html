中位数与顺序统计量

在一个由$n$个元素组成的集合中，第$i$个顺序统计量（$ order statistic $）是该集合中第$i$小的元素。例如，在一个元素集合中，最小值是第1个顺序统计量$(i = 1)$，最大值是第$n$个顺序统计量$(i = n)$。

中位数是它所属集合的“中点元素”。当$n$为奇数时，中位数是唯一的。当$n$为偶数时，存在两个中位数，分别位于$i = n/2$和$i = n/2 + 1$处。

接下来我们讨论一个由$n$个互异的元素构成的集合中选择第$i$个顺序统计量的问题。对于这一问题形式化定义为如下的选择问题:

<strong>输入：</strong>一个包含$n$个（互异的）数的集合$A$和一个整数$i, 1 \leq i \leq n$。

<strong>输出：</strong>元素$x \in A$，且$A$中恰好有$i - 1$个其他元素小于它。

通过选择合适的排序算法，我们可以在有限时间内快速解决这个问题(堆排序或归并排序$O(lgn)$)。这里学习一些更快的算法。

最大值和最小值问题

对于一个具有$n$个元素的数组，我们可以在比较最多$n-1$次的情况下，寻找到最小值或最大值。一种更优的算法同时寻找最大值和最小值。

<strong>    同时寻找最大值和最小值</strong>

如果独立的去寻找最大值和最小值，需要$(n - 1) + (n - 1) = 2n - 2$次比较。事实上，我们可以最多$3\times \lfloor n/ 2 \rfloor$次比较就可以同时寻找最大值和最小值。具体的如下：每次对两个数进行比较，将较大的值与已知的最大值比较，将较小值与已知的最小值比较。这样每次我们仅仅需要3次比较，而总共需要比较的数对有$n/2$对，所以总共的比较次数为$3\times \lfloor n/ 2 \rfloor$。

对于如何得到最初的最大值与最小值呢？实际上开始的最大值与最小值的设定依赖于$n$是偶数还是奇数，当$n$为奇数时，最大值与最小值相等，均为第一个数值。当$n$为偶数时，将前两个元素进行一次比较，确定最大值与最小值的初始值。

期望为线性时间的选择算法

接下来我们要学习一种解决选择问题的分治算法。$RANDOMIZED-SELECT$是以快速排序算法为模型的。该算法利用了$RANDOMIZED-PARTITION$过程。

该算法使用随机分割将待寻找部分分割为2部分，其中该数组中比分割点小的元素，移动到分割点之前；该数组中比分割点大的元素，移动到分割点之后。之后进行下一次寻找。期望时间复杂度$O(n)$，最坏时间复杂度$O(n……2)$.

最坏时间为线性时间的选择算法

现在我们来看一个最坏情况运行时间为$O(n)$的选择算法：$SELECT$。和$RANDOMIZED-SELECT$一样，$SELECT$算法也是通过对输入数组的递归划分来找出所需元素，但是，在该算法中能够保证对数组的一个好的划分。$SELECT$算法使用的是来自快速排序的确定性划分算法$PARTITION$，但做出了修改，将划分的主元也作为输入参数。

通过执行下列步骤，算法$SELECT$可以确定一个有$n>1$个不同元素的输入数组中第$i$小的元素。（如果$n = 1$，则$SELECT$只返回它的唯一输入数值作为第$i$小的元素。）

  1. 将输入数组的$n$个元素划分为$\lfloor n/5 \rfloor$组，每组5个元素，且至多只有一组由剩下的$n mod 5$个元素组成。
  2. 寻找这$\lceil n/5 \rceil$组中每一组的中位数：首先对每组元素进行插入排序，然后确定每组有序元素的中位数。
  3. 对第二步中找到的$\lceil n/5 \rceil $个中位数，递归调用$SELECT$以找出其中位数$x$（如果有偶数个中位数，为了方便，约定$x$是较小的中位数）。
  4. 利用修改过的$PARTITION$版本，按中位数的中位数$x$对输入数组进行划分。让$k$比划分的低区中的元素数目多1，因此$x$是第$k$小的元素，并且有$n-k$个元素在划分的高区。
  5. 如果$i = k$，则返回$x$。如果$ik$则在高区中递归查找第$i-k$小的元素。
