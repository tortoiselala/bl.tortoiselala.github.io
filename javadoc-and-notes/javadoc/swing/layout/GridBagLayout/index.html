java swing 复杂的布局管理器-网格组布局(grid bag layout)

网格组布局是所有布局管理器之母。可以将网格组布局看成是没有任何限制的网格布局。在网格组布局中，行和列的尺寸可以改变。可以将相邻的单元合并以适应较大的组件（类似于很多字处理器以及HTML都使用这个功能编辑表格：一旦需要就合并相邻单元格）。组件不需要填充整个单元格区域，并可以指定在单元格内的对齐方式。

网格组管理器的使用方式

  1. 建立一个GridBagLayout的对象。不需要指定网格的行数和列数。布局管理器会根据后面的信息猜测出来。

  2. 将GridBagLayout对象设置成组件的布局管理器。

  3. 为每个组件建立一个GridBagConstraints对象。设置GridBagConstraints对象的域以便指出组件在网格组中的布局方案。

    设置GridBagdConstrints对象的状态约束

    1、grdidx、gridy、 gridwidth、grdiheight参数

    这些约束定义了组件在网格中的位置。gridx和gridy指定了被添加组件左上角的行、列位置。gridwidth和gridweight指定了组件占据的行数和列数。

    网格的坐标从0开始。grid=0grid=0和gridy=0gridy=0 代表最左上角。eg:gridx=2和gridy=0eg:gridx=2和gridy=0代表起始位置为0行2列，gridwidth=1和gridheight=4gridwidth=1和gridheight=4代表横跨了4行1列。

    2、增量域

    在网格布局中需要为每个区域设置增量域(weightx和weighty)(weightx和weighty)。如果将增量设置为0，则这个区域永远为初始尺寸。如果将所有的区域增量都设置为0，则容器就会聚集在为他分配的区域中间，而不会通过拉伸来填充它。

    值得注意的是，增量并不给出列的相对大小。当容器超过首选大小时，增量表示分配给每个区域的扩展比例值。

    3、fill和ancher参数

    如果不希望组件拉伸至整个区域，就需要设置fill约束。它有四个有效值：GridBagConstraints.NONE、GridBagConstraints.NONE、GridBagConstraints.HORIZONTAL、GridBagConstraints.HORIZONTAL、GridBagConstraints.VERTICALGridBagConstraints.VERTICAL和GridBagConstraints.BOTHGridBagConstraints.BOTH。

    • NONENONE# 组件不会做任何变化
    • HORIZONTALHORIZONTAL组件的宽度变化，使得水平填充其显示区域，高度不变化
    • VERTICALVERTICAL组件的高度变化，使得垂直填充其显示区域，宽度不变化
    • BOTHBOTH组件的高度和宽度均变化，以填充其显示区域

    如果组件没有填充整个区域，可以通过设置anchor域来指定其位置。有效值为GridBagConstraints.NORTHWESTGridBagConstraints.NORTHWEST、GridBagConstraints.NORTH、GridBagConstraints.NORTH、GridBagConstraints.NORTHEAST、GridBagConstraints.NORTHEAST、GridBagConstraints.WESTGridBagConstraints.WEST、GridBagConstraints.CENTER（默认值)GridBagConstraints.CENTER（默认值)、GridBagConstraints.EASTGridBagConstraints.EAST、GridBagConstraints.SOUTHWESTGridBagConstraints.SOUTHWEST、GridBagConstraints.SOUTHGridBagConstraints.SOUTH和GridBagConstraints.SOUTHEASTGridBagConstraints.SOUTHEAST

    4、填充

    可以通过设置GridBagLayoutGridBagLayout的insert域在组件周围增加附加的空白区域。通过设置Insets对象的left、top、right和buttom指定组件周围的空间量。这被称为外部填充（或外边距）(external padding)。

    通过设置ipadx和ipady指定内部填充(内边距)(internal padding)。这两个值被加到组件的最小宽度和最小高度上。这样可以保证每个组件不会收缩至最小尺寸之下。

    5、指定gridx,gridy,gridwidthgridx,gridy,gridwidth和gridheightgridheight参数的另一种方法

    AWT文档建议不要将gridx和gridygridx和gridy设置为绝对位置，应该将它们设置为常量GridBagConstraints.RELATIVEGridBagConstraints.RELATIVE。然后，按照标准顺序将组件添加到网格布局中。

    还需要通过gridwidth和gridheightgridwidth和gridheight域指定一个适当的值来设置组件横跨的行数和列数。除此之外，如果组件扩展至最后一行或最后一列，则不要给出一个实际的数值，而是用常量GridBagConstraints.REMAINDERGridBagConstraints.REMAINDER替代，这样会告诉布局管理器这个组件是本行上最后一个组件。

    • 将所有的增量设置为100.如果需要某行或某列时钟保持默认的大小，就把这行或这列中所有的组件和weightx和weighty设置为0。weightx和weighty设置为0。

API

java.awt.GridBagConstraints

  • int gridx, gridy–指定单元格的起始行和列，默认为0

  • int gridwidth, gridheight–指定单元格的行和列的范围，默认值为1

  • double weightx, weighty–指定单元格扩大时的容量，默认为0

  • int ancher–表示组件在单元格中的对齐方式。可以选择的绝对位置有

    | NORTHWESTNORTHWEST | NORTHNORTH | NORTHEASTNORTHEAST |
    | :————-: | :———-: | :————-: |
    | WESTWEST | CENTERCENTER | EASTEAST |
    | SOUTHWESTSOUTHWEST | SOUTHSOUTH | SOUTHEASTSOUTHEAST |

    或者各个方向上的相对位置

    | FIRST_LINE_START | LINE_START | FIRST_LINE_END |
    | :—————–: | :———-: | :—————: |
    | PAGE_START | CENTER | PAGE_END |
    | LAST_LINE_START | LINE_END | LAST_LINE_END |
    如果应用程序有可能从右向左，或者自顶至底排列文本。就应该使用后者，默认值为CENTER。

  • int fill–指定组件在单元格中的填充行为取值为NONE、BOTH、HORIZONTAL，或者VERTICAL。默认值为NONE。

  • int ipadx, ipady–指定组件周围的“内部”填充，默认值为0.

  • Insets insets–指定组件周围的“外部”填充。默认为不填充。
